## Version Format

Versions are tagged as: `vMAJOR.MINOR.PATCH`

Example: `v1.0.0` represents the first major release with no minor or patch changes.

## Release Cycle

- **Major Releases**: Breaking changes to core axioms or architecture
- **Minor Releases**: New modules, features, or specifications (quarterly)
- **Patch Releases**: Documentation updates, clarifications, fixes (as needed)

## Documentation Standards

All changes must include:
1. Description of the change
2. Reason for the change
3. Impact assessment
4. Migration instructions (if applicable)

## Change Categories

- **Added**: New features or content
- **Changed**: Existing functionality modifications
- **Deprecated**: Soon-to-be-removed features
- **Removed**: Deleted features
- **Fixed**: Bug fixes
- **Security**: Vulnerability patches
